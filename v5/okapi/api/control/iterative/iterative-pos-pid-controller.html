okapi::IterativePosPIDController¶

An IterativePositionController that uses the PID algorithm. If you are trying to create an instance of this class, you should most likely be using the IterativeControllerFactory instead of a constructor from this class.

Constructor(s)¶

Prototype
IterativePosPIDController(double ikP, double ikI, double ikD, double ikBias,
                          const TimeUtil &itimeUtil,
                          std::unique_ptr<Filter> iderivativeFilter = std::make_unique<PassthroughFilter>())
Parameters  
ikp The P term gain.
ikI The I term gain.
ikD The D term gain.
ikBias The controller bias.
itimeUtil See TimeUtil docs.
iderivativeFilter The filter to use for filtering the derivative term.
Prototype
IterativePosPIDController(const Gains &igains, const TimeUtil &itimeUtil,

Parameters  
igains See Gains below.
itimeUtil See TimeUtil docs.
iderivativeFilter The filter to use for filtering the derivative term.

step¶

Do one iteration of the controller. Outputs in the range [-1, 1].

Prototype
double step(double ireading) override
Parameters  
ireading The new sensor reading.

Returns: The controller output.

setTarget¶

Sets the target for the controller.

Prototype
void setTarget(double itarget) override
Parameters  
itarget The new target.

controllerSet¶

Writes the value of the controller output. This method might be automatically called in another thread by the controller. The range of input values is expected to be [-1, 1].

Prototype
void controllerSet(double ivalue) override
Parameters  
ivalue The controller’s output in the range [-1, 1].

getTarget¶

Gets the last set target, or the default target if none was set.

Prototype
double getTarget() override

Returns: The last target.

getOutput¶

Returns the last calculated output of the controller. Default is 0. Output is in the range [-1, 1] by default.

Prototype
double getOutput() const override

Returns: The previous output from the filter.

getError¶

Returns the last error of the controller.

Prototype
double getError() const override

Returns: The last error of the controller.

isSettled¶

Returns whether the controller has settled at the target. Setting is when the error or derivative of error has been small enough for a long enough period.

If the controller is disabled, this method must return true.

Prototype
bool isSettled() override

Returns: Whether the controller is settled.

setErrorSumLimits¶

Sets the error sum bounds. Default bounds are [0, std::numeric_limits<double>::max()]. Error will only be added to the integral term when its absolute value between these bounds of either side of the target.

Prototype
virtual void setErrorSumLimits(double imax, double imin)
Parameters  
imax The upper bound.
imin The lower bound.

reset¶

Resets the controller’s internal state so it is similar to when it was first initialized, while keeping any user-configured information. This implementation also stops movement.

Prototype
void reset() override

Prototype

Parameters  
iresetOnZero Whether the integrator should be reset when error is 0 or changes sign.

flipDisable¶

Changes whether the controller is off or on. Turning the controller on after it was off will cause the controller to move to its last set target, unless it was reset in that time.

Prototype
void flipDisable() override

Prototype
void flipDisable(bool iisDisabled) override
